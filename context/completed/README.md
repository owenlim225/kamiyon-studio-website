# Completed work archive

Long-term history for finished milestones. One markdown file per closed phase or milestone.

**Active work lives in** [`../progress-tracker.md`](../progress-tracker.md). Do not treat this folder as the live tracker.

## Archival process

When a task/phase is marked complete:

1. Create `context/completed/YYYY-MM-DD-<slug>.md` with the finished context (goal, decisions, checklist, verification results, session notes).
2. Add a row to the table below.
3. Remove the bulky finished block from `progress-tracker.md` (at most a one-line “Done — see completed/…” pointer).
4. Relocate historical detail — do not delete it.
